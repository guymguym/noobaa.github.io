Starting from NooBaa V2, you can connect your application right after system activation. 

For earlier versions, you must follow the guidelines for introducing storage resource first.
 
A native access via AWS S3 compatible application is a REST based protocol commonly used by AWS S3 compatible clients (e.g. CloudBerry). By connecting an application to NooBaa you can start writing and reading data without accessing the management UI.
 
Steps:
  1. Go to Overview > Connect application
  2. Select the type of connection your application uses, Native access for S3 client.
  3. Choose an account you want to connect with (each account with S3 access permission has different keys set and can be used for application connection).

  4. Open the application you wish to connect and click on enter new account (note that each application has a different term for this action)
  5. Choose Storage type as S3 compatible storage and copy the REST endpoint from the UI. Paste it to REST endpoint field in your application

  6. Copy The Access Key and secret key and paste it into the access key field in your application
  7. Apply the changes in your S3 application and test the connection by uploading a file to a bucket.
All files will appear in Buckets > Selected Bucket > files list
