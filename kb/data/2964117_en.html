Using Restic with NooBaa requires the following steps:

Configuration
 
export RESTIC_REPOSITORY="s3:https://<NooBaa endpoint>/<NooBaa bucket>” 
export AWS_ACCESS_KEY_ID="<NooBaa access key>” 
export AWS_SECRET_ACCESS_KEY="<NooBaa secret key>” 
export RESTIC_PASSWORD="<preferred password> ” 

SSL certificate

Restic requires a valid SSL certificate on the target backup. 
If you are interested in uploading a valid certificate to your NooBaa system, please read the following article.
Alternatively, you can provide CA certificate for Restic with the flag --cacert and provide a self-signed certificate for both NooBaa and Restic. 

Example:
let's assume that NooBaa DNS name is www.testme.com. 
You can generate a self-signed certificate on the following website: http://www.selfsignedcertificate.com/
Download the 37665497_www.testme.com.cert and 37665497_www.testme.com.key files, zip them and upload NooBaa per the instructions in the following article.

Running Restic

restic --cacert 37665497_www.testme.com.cert init --repo  s3:https://www.testme.com/restic  
restic --cacert 37665497_www.testme.com.cert  -r s3:https://www.testme.com/restic backup .
