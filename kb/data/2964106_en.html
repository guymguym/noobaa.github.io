Using Duplicity (http://duplicity.nongnu.org/) with NooBaa requires the following steps:

Configure /etc/boto.cfg
Make sure you have the following lines. 

[Credentials]
aws_access_key_id = <NooBaa access key id>
aws_secret_access_key = <NooBaa secret access key>
[Boto]
# If using a valid SSL certificate on NooBaa, set to True
is_secure = False

Verify that you don't have any conflict with  ~/.aws/credentials

~/.aws/credentials may contain [default] section that will take priority on boto.cfg.
In such a case, you better change the default to a named profile.

Note: remember to change the applications that use ~/.aws/credentials to use the newly named profile. 

Run

duplicity  .  "s3://<NooBaa rest endpoint>/<target bucket>"

When listing NooBaa bucket, you should see similar objects list
