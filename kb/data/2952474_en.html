Regions in NooBaa is a location tag.
The regions configuration allows customers to associate resources with location.
Associate a region with location helps to improve performance by setting locality awareness while taking data placement decision and reading data. In addition, using regions is another way to reduce egress costs due to locality awareness during data consumption activities. 

How does it work? 

Resources within the same region will be prioritized for data assignments and for serving read requests.
Using a local endpoint in this region will benefit the proximity to the local resources by reading and writing the data directly from local resources, if possible, and mirror data to a remote location after achieving the required resiliency level locally. 
Another example is using an endpoint in the cloud provider network, and associate both the endpoint and the cloud resource in NooBaa with a region, so applications that run in the cloud would use the right endpoint to avoid redundant egress cost. 

How to use regions in NooBaa management console? 

In order to use regions in NooBaa system, you need to assign regions to the resources.

Steps:
1. Go to Resources > Node pools or cloud storage resource page 



2. In the resource page, click on Assign Region


3. Type the name of the resource region as you decide. To add a new region hit Enter key, if you wish to use an existing region (from another resource), start typing and choose the wanted option from the suggestion box. The region name can be anything you consider as a region (Continent, city, building etc.).


Once the region is set, you can connect the resources of the same region as data placement for data buckets.
