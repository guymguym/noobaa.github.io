Starting from NooBaa V2, the system includes bucket spillover which can be used for testing and as a temporary storage.

In order to move ahead and build a scalable system, the first step would be introducing storage resource to your NooBaa system.

This step is required for writing and reading data.

There are two types of storage resources in NooBaa:
 
1. Linux or Windows server with more than 15GB free storage. You will need at least three servers, each with at least 15GB free storage. 
2. A native cloud storage, such as AWS S3 bucket or Azure Blob storage.

Once you have the resources in place, connect an application and start working.

Adding Linux or Windows servers can be done from the main screen, by clicking on the Install Nodes button
Follow the wizard instructions, copy the one-liner and run it on each of the three servers.





Adding cloud resource storage can be done by clicking on the resource icon -> Cloud Storage tab -> Add Cloud Resource button.  



If you didn't connect your application already, let's move to the next step - Connect your application
